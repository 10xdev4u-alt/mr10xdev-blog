5 ways to make an agent feel like a teammate: greeting,
owning, explaining, learning, growing. The small touches
that make the agent a colleague. The line between "tool"
and "teammate."

## Touch 1: Greeting

The agent greets the user. The agent says hello. The
agent is friendly.

**Bad:** the agent is silent. The agent doesn't greet.
The agent is cold.

**Good:** the agent says "Hi! I'm the triage agent. How
can I help?" The agent is friendly. The agent is a
teammate.

The greeting is the difference between "the agent is a
tool" and "the agent is a colleague."

## Touch 2: Owning

The agent owns the task. The agent follows up. The
agent takes responsibility.

**Bad:** the agent does the task and forgets. The agent
doesn't follow up. The agent is unreliable.

**Good:** the agent does the task and follows up. The
agent says "I labeled it as bug. Want me to also assign
it to alice?" The agent is reliable.

The owning is the difference between "the agent is a
contractor" and "the agent is an owner."

## Touch 3: Explaining

The agent explains its decisions. The agent shows its
work. The agent is transparent.

**Bad:** the agent does the task silently. The user
doesn't know why. The user is suspicious.

**Good:** the agent says "I labeled it as bug because the
issue describes a crash." The user understands. The user
trusts.

The explaining is the difference between "the agent is a
black box" and "the agent is a colleague."

## Touch 4: Learning

The agent learns from feedback. The agent improves. The
agent grows.

**Bad:** the agent does the same thing wrong forever. The
agent doesn't learn. The agent is frustrating.

**Good:** the agent learns from the user's feedback. The
agent says "Thanks for the feedback. I'll remember that
for next time." The agent grows.

The learning is the difference between "the agent is
static" and "the agent is alive."

## Touch 5: Growing

The agent's capabilities grow over time. The agent learns
new skills. The agent becomes more useful.

**Bad:** the agent does the same thing forever. The agent
is limited. The agent is a one-trick pony.

**Good:** the agent learns new tools. The agent handles
new cases. The agent is a generalist.

The growing is the difference between "the agent is a
toy" and "the agent is a teammate."

## The 5 together

The 5 are the teammate. The teammate is the colleague.
The colleague is the value.

| Touch | What it ensures |
|---|---|
| Greeting | The user is welcomed |
| Owning | The user can rely |
| Explaining | The user can trust |
| Learning | The user can correct |
| Growing | The user can grow |

The agent that has all 5 is the teammate. The agent that
has 1 is the tool. The teammate is adopted. The tool is
not.

## The 80/20

80% of the value comes from:
- Owning (the user can rely)
- Explaining (the user can trust)

20% comes from:
- Greeting (the user is welcomed)
- Learning (the user can correct)
- Growing (the user can grow)

Focus on the 80% first. Add the 20% as you grow.

## The test

The agent feels like a teammate if:
- A user feels welcomed on first contact
- A user can rely on the agent to follow up
- A user can understand why the agent decided
- A user can correct the agent
- A user can see the agent grow

If any of these fails, the agent feels like a tool. Fix
the agent.

## The lesson

5 touches. 1 teammate. 1 lesson: invest in the small
touches.

The agent that invests in the small touches is the
teammate. The agent that doesn't is the tool. The
teammate is adopted. The tool is not.

The agent era is here. The teammate is the design. The
design is the discipline. The discipline is the value.
