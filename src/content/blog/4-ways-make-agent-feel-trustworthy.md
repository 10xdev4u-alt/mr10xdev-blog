4 ways to make an agent feel trustworthy: accurate,
consistent, transparent, accountable. The principles that
build trust. The line between "scary" and "trusted."

## Principle 1: Accurate

The agent is right. The agent is correct. The agent
doesn't make mistakes. The agent is reliable.

**Bad:** the agent is right 50% of the time. The user
doesn't trust. The user double-checks.

**Good:** the agent is right 95% of the time. The user
trusts. The user delegates.

The accurate is the difference between "let me check" and
"I'll trust it."

## Principle 2: Consistent

The agent is the same every time. The agent is
deterministic. The agent is predictable.

**Bad:** the agent labels the same issue differently each
time. The user is confused. The user doesn't trust.

**Good:** the agent labels the same issue the same way
every time. The user trusts. The user delegates.

The consistent is the difference between "I have to
re-check" and "I trust the result."

## Principle 3: Transparent

The agent shows its work. The agent is auditable. The
agent is debuggable.

**Bad:** the agent is a black box. The user can't see how
it decided. The user has to trust blindly.

**Good:** the agent shows its reasoning. The user can
verify. The user can correct.

The transparent is the difference between "trust me" and
"here's what I did."

## Principle 4: Accountable

The agent has an owner. The agent has a maintainer. The
agent has a feedback channel.

**Bad:** the agent has no owner. The user can't get
help. The user gives up.

**Good:** the agent has a maintainer. The user can ask
questions. The user can report bugs.

The accountable is the difference between "who do I
contact?" and "I'll email the maintainer."

## The 4 together

The 4 are the trust. The trust is the value. The value is
the adoption.

| Principle | What it ensures |
|---|---|
| Accurate | The user can delegate |
| Consistent | The user can rely |
| Transparent | The user can verify |
| Accountable | The user can get help |

The agent that has all 4 is the trusted agent. The agent
that has 1 is the untrusted agent. The trusted agent is
adopted. The untrusted agent is not.

## The 80/20

80% of the value comes from:
- Accurate (the user can delegate)
- Consistent (the user can rely)

20% comes from:
- Transparent (the user can verify)
- Accountable (the user can get help)

Focus on the 80% first. Add the 20% as you grow.

## The test

The agent feels trustworthy if:
- A user can rely on the output
- A user gets the same output for the same input
- A user can see why the agent decided
- A user can contact the maintainer

If any of these fails, the agent feels scary. Fix the
agent.

## The lesson

4 principles. 1 trust. 1 lesson: invest in the principles.

The agent that invests in the principles is the trusted
agent. The agent that doesn't is the untrusted agent. The
trusted agent is adopted. The untrusted agent is not.

The agent era is here. The trust is the design. The design
is the discipline. The discipline is the value.
