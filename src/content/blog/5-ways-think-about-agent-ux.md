5 ways to think about agent UX: discoverable, predictable,
helpful, transparent, respectful. The principles that make
agents feel good. The framework, the examples, the lesson.

## Principle 1: Discoverable

The user can find the agent's features. The user doesn't
have to read the docs. The features are obvious.

**Bad:** the agent has 10 features. The user uses 1. The
user doesn't know the other 9 exist.

**Good:** the agent has 10 features. The features are in
the menu. The features are discoverable. The user uses 5.

The discoverable is the difference between "the agent has
features" and "the user uses the features."

## Principle 2: Predictable

The user knows what the agent will do. The user can predict
the output. The user is in control.

**Bad:** the agent does something unexpected. The user is
surprised. The user is angry.

**Good:** the agent does what the user expects. The user is
happy. The user is in control.

The predictable is the difference between "what did it
do?" and "exactly what I asked for."

## Principle 3: Helpful

The agent is actually useful. The agent saves the user
time. The agent is a force multiplier.

**Bad:** the agent is a toy. The agent doesn't save time.
The agent is a gimmick.

**Good:** the agent saves the user 30 minutes a day. The
agent is essential. The agent is a tool.

The helpful is the difference between "the agent is cool"
and "the agent is essential."

## Principle 4: Transparent

The user knows what the agent is doing. The user knows
why. The user can verify.

**Bad:** the agent is a black box. The user has to trust.
The user can't verify.

**Good:** the agent shows its work. The user can verify. The
user can correct.

The transparent is the difference between "trust me" and
"here's what I did."

## Principle 5: Respectful

The agent respects the user. The agent doesn't waste the
user's time. The agent doesn't insult the user.

**Bad:** the agent makes the user feel stupid. The agent
wastes the user's time. The agent is rude.

**Good:** the agent treats the user with respect. The
agent values the user's time. The agent is polite.

The respectful is the difference between "ugh, the agent"
and "thank you, agent."

## The 5 together

The 5 are the UX. The UX is the experience. The
experience is the value.

| Principle | What it ensures |
|---|---|
| Discoverable | The user can find |
| Predictable | The user can plan |
| Helpful | The user can rely |
| Transparent | The user can trust |
| Respectful | The user can stay |

The agent that has all 5 is the well-designed agent. The
agent that has 1 is the poorly-designed agent. The
well-designed agent is adopted. The poorly-designed agent
is not.

## The 80/20

80% of the value comes from:
- Predictable (the user trusts)
- Helpful (the user relies)

20% comes from:
- Discoverable (the user finds)
- Transparent (the user verifies)
- Respectful (the user stays)

Focus on the 80% first. Add the 20% as you grow.

## The test

The agent feels good if:
- A new user can find a feature in 30 seconds
- A new user can predict the output
- A new user can save 5 minutes on the first task
- A new user can verify the output
- A new user feels respected

If any of these fails, the agent feels bad. Fix the agent.

## The lesson

5 principles. 1 UX. 1 lesson: design for the user.

The agent that designs for the user is the well-designed
agent. The agent that designs for itself is the
poorly-designed agent. The well-designed agent is
adopted. The poorly-designed agent is not.

The agent era is here. The UX is the design. The design is
the discipline. The discipline is the value.
