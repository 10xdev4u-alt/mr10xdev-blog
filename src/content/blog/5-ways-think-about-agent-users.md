5 ways to think about users: developer, team, enterprise,
hobbyist, executive. Each is a different user with different
needs. The framework, the examples, the lesson.

## User 1: Developer

The user is a developer. The user codes. The user wants
power.

**Examples:**
- The user reads the docs
- The user runs the agent in the terminal
- The user customizes the agent

**When to use:** the user is technical. The user is
hands-on. The user wants control.

**Pros:**
- Powerful (the developer wants power)
- Specific (the developer is specific)
- Engaged (the developer is engaged)

**Cons:**
- Friction (the developer has friction)
- Effort (the developer takes effort to onboard)
- Limit (the developer is a small audience)

## User 2: Team

The user is a team. The user collaborates. The user
wants shared value.

**Examples:**
- The team uses the agent for triage
- The team shares the agent's memory
- The team delegates to the agent

**When to use:** the user is a team. The user is
collaborative. The user wants shared.

**Pros:**
- Shared (the team is shared)
- Compound (the team compounds)
- Aligned (the team is aligned)

**Cons:**
- Coordination (the team has coordination)
- Coupled (the team is coupled to the agent)
- Limit (the team is a medium audience)

## User 3: Enterprise

The user is an enterprise. The user is large. The user
wants custom.

**Examples:**
- The enterprise buys the agent
- The enterprise customizes the agent
- The enterprise supports the agent

**When to use:** the user is enterprise. The user is
custom. The user is large.

**Pros:**
- Revenue (the enterprise is high revenue)
- Custom (the enterprise is custom)
- Long-term (the enterprise is long-term)

**Cons:**
- Sales (the enterprise needs sales)
- Complexity (the enterprise is complex)
- Slow (the enterprise is slow to close)

## User 4: Hobbyist

The user is a hobbyist. The user is curious. The user
wants to learn.

**Examples:**
- The hobbyist tries the agent
- The hobbyist gives feedback
- The hobbyist shares the agent

**When to use:** the user is a hobbyist. The user is
curious. The user is engaged.

**Pros:**
- Engaged (the hobbyist is engaged)
- Word-of-mouth (the hobbyist spreads the word)
- Feedback (the hobbyist gives feedback)

**Cons:**
- No revenue (the hobbyist doesn't pay)
- Variable (the hobbyist is variable)
- Limit (the hobbyist is a small audience)

## User 5: Executive

The user is an executive. The user is busy. The user
wants results.

**Examples:**
- The executive uses the agent for summaries
- The executive gets the agent's reports
- The executive delegates to the agent

**When to use:** the user is an executive. The user is
busy. The user wants results.

**Pros:**
- Revenue (the executive is high revenue)
- Results (the executive wants results)
- Word-of-mouth (the executive spreads the word)

**Cons:**
- Specific (the executive is specific)
- Limited (the executive is a small audience)
- Effort (the executive is hard to reach)

## The 5 together

The 5 are the users. The users are the audience. The
audience is the value.

| User | Effort | Revenue | Best for |
|---|---|---|---|
| Developer | High | Low | Power |
| Team | Medium | Medium | Shared |
| Enterprise | High | High | Custom |
| Hobbyist | Low | None | Engagement |
| Executive | Low | High | Results |

The user that matches the agent is the right user.

## The 80/20

80% of the value comes from:
- Developer (the user is technical)
- Team (the user is collaborative)

20% comes from:
- Enterprise (the user is large)
- Hobbyist (the user is curious)
- Executive (the user is busy)

Focus on the 80% first. Add the 20% as you grow.

## The choice

For each agent, ask:
- Is the user technical? (developer)
- Is the user collaborative? (team)
- Is the user large? (enterprise)
- Is the user curious? (hobbyist)
- Is the user busy? (executive)

The right answer is the right user at the right cost.

## The lesson

5 users. 1 user. 1 lesson: pick the right one.

The user that matches the agent is the right user. The
user that doesn't match is the wrong user.

The agent era is here. The user is the design. The
design is the choice. The choice is the discipline.
