I get asked a lot: "how do you write so much so fast?" Here's the actual
template I use for every post on this blog.

## 1. Start with the takeaway

One sentence. The thing you want the reader to remember. If you can't
write it in 30 seconds, you don't have a post yet.

Examples from my drafts:
- "Ship small, ship often, ship ugly."
- "Git is the only piece of infrastructure with a 15-year audit trail."
- "The agent is a file. The memory is a directory. The PR is the review."

The takeaway is the title's soul. The title is the takeaway with a
verb.

## 2. The 5-section structure

Most of my posts follow this:

1. **The hook** (3-5 lines) — why this matters right now
2. **The problem** (1-2 paragraphs) — what was broken or missing
3. **The fix** (3-5 paragraphs with code) — the actual solution
4. **The numbers** (1 paragraph) — the proof it worked
5. **The takeaway** (3-5 lines) — restate the title in different words

This isn't the only structure, but it's the default. If you're stuck,
fall back to it.

## 3. Code is the best prose

A code block in a blog post is worth 5 paragraphs. If you can show
the API, show the API. If you can show the config, show the config.
If you can show the bug + the fix, show the diff.

My post on Husk v0.6.0 has 4 code blocks. The word count is ~800. The
code count is ~50 lines. Ratio: 16 words per line of code. That's a
readable post.

## 4. Voice: 4 rules

1. **Contractions always.** "Don't" not "do not". "It's" not "it is".
2. **Short sentences win.** Mix 8-word and 24-word sentences.
3. **Em dashes for asides.** — like this. (Or commas if you must.)
4. **No "we" or "you" unless you're talking to a specific person.**
   "I built X" not "We built X".

## 5. The 80/20 of voice

80% of voice is word choice. Drop the academic vocabulary. "Utilize" → "use".
"Facilitate" → "help". "Subsequently" → "next". "The implementation" → "the code".

The other 20% is rhythm. Read your post out loud. If you stumble, the
reader will too.

## 6. The 10-minute test

Before publishing, ask: would I read this in 10 minutes? If no, cut.
Most of my posts take 6-8 minutes to read. If it's longer than 12, I
split it into two posts.

## 7. The publish reflex

The biggest enemy of blog posts is "I'll publish after one more
edit." The blog post you publish today is better than the perfect
post you publish next month.

My rule: **a good-enough post today beats a perfect post never.**

Ship it. Fix it in comments.
