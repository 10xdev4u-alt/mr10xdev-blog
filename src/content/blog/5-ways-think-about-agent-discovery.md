5 ways users discover agents: search, browse, recommend,
share, import. Each is a different discovery lever. The
framework, the examples, the lesson.

## Way 1: Search

The user searches. The user types a query. The user finds
the agent.

**Examples:**
- "triage agent" → the triage agent
- "doc agent" → the doc agent
- "release agent" → the release agent

**When to use:** the user knows what they want. The user
is looking for a specific thing. The user is targeted.

**Pros:**
- Targeted (the user finds exactly what they want)
- High-intent (the user is ready to use)
- Fast (the user gets the result in 1 second)

**Cons:**
- Friction (the user has to know what to search)
- SEO (the agent has to be optimized for search)
- Competition (the user has many results)

## Way 2: Browse

The user browses. The user scrolls. The user finds the
agent.

**Examples:**
- The marketplace of agents
- The category page (e.g., "Triage agents")
- The "popular" page

**When to use:** the user is exploring. The user is open
to suggestions. The user is browsing.

**Pros:**
- Discovery (the user finds things they didn't know)
- Browsing (the user can take their time)
- Categories (the user can filter by use case)

**Cons:**
- Distraction (the user can get lost)
- Time (the user has to spend time)
- Quality (the user has to filter)

## Way 3: Recommend

The user is recommended. The system suggests. The user
finds the agent.

**Examples:**
- "You might also like: doc agent"
- "Other users also used: release agent"
- "Trending: triage agent"

**When to use:** the user is open to suggestions. The
user trusts the system. The user is browsing.

**Pros:**
- Personal (the recommendation is for the user)
- Relevant (the recommendation matches the user)
- Discovery (the user finds new agents)

**Cons:**
- Filter bubble (the user only sees similar things)
- Cold start (the system doesn't know the user)
- Quality (the recommendation can be wrong)

## Way 4: Share

The user shares. The user tells others. The user spreads
the agent.

**Examples:**
- "Check out this triage agent"
- "I use this doc agent, you should too"
- "Here's the link to the release agent"

**When to use:** the user loves the agent. The user wants
to share. The user is an advocate.

**Pros:**
- Trusted (the user trusts their friends)
- Viral (the user can spread the word)
- High-quality (the user only shares what they love)

**Cons:**
- Slow (the user has to share)
- Limited (the user has a small network)
- Inconsistent (the user might not share)

## Way 5: Import

The user imports. The user copies. The user uses the
agent.

**Examples:**
- The user copies the manifest from the marketplace
- The user imports the agent from a template
- The user installs the agent from npm

**When to use:** the user is committed. The user is
ready. The user wants to use.

**Pros:**
- Easy (the user just copies)
- Fast (the user is using in 1 minute)
- Standard (the user uses the same import everywhere)

**Cons:**
- Friction (the user has to import)
- Setup (the user has to set up the agent)
- Compatibility (the user might have version issues)

## The 5 together

The 5 are the discovery. The discovery is the adoption.
The adoption is the growth.

| Way | Friction | Intent | Best for |
|---|---|---|---|
| Search | High | High | Targeted users |
| Browse | Low | Medium | Explorers |
| Recommend | Low | Medium | Browsers |
| Share | Low | High | Advocates |
| Import | High | High | Committed |

The way that matches the user is the right way.

## The 80/20

80% of the value comes from:
- Search (the user is targeted)
- Import (the user is committed)

20% comes from:
- Browse (the user is exploring)
- Recommend (the user is browsing)
- Share (the user is advocating)

Focus on the 80% first. Add the 20% as you grow.

## The choice

For each agent, ask:
- Is the user targeted? (search)
- Is the user exploring? (browse)
- Is the user browsing? (recommend)
- Is the user advocating? (share)
- Is the user committed? (import)

The right answer is the right way at the right friction.

## The lesson

5 ways. 1 discovery. 1 lesson: support all 5.

The agent that supports all 5 is the discoverable agent.
The agent that supports 1 is the invisible agent. The
discoverable agent is adopted. The invisible agent is
not.

The agent era is here. The discovery is the design. The
design is the choice. The choice is the discipline.
