3 ways to think about the future: features, ecosystem,
category. Each is a different future. The framework, the
examples, the lesson.

## Future 1: Features

The future is features. The agent will have more
features. The agent will be more capable.

**Examples:**
- "v0.5.0 will add memory"
- "v1.0.0 will add streaming"
- "v2.0.0 will add multi-agent"

**When to use:** the user is asking about features. The
user wants more. The maintainer is shipping.

**Pros:**
- Predictable (the features are predictable)
- Shippable (the features are shippable)
- Standard (the features are standard)

**Cons:**
- Limited (the features are limited)
- Coupled (the features are coupled to the agent)
- Drift (the features can drift)

## Future 2: Ecosystem

The future is ecosystem. The agent will have a community.
The agent will have plugins.

**Examples:**
- "We'll have a community of contributors"
- "We'll have a marketplace of plugins"
- "We'll have a standard for agent manifests"

**When to use:** the user is asking about community. The
user wants to participate. The maintainer is building.

**Pros:**
- Community (the ecosystem has a community)
- Network (the ecosystem has a network effect)
- Standard (the ecosystem is a standard)

**Cons:**
- Slow (the ecosystem is slow to build)
- Quality (the ecosystem can have quality issues)
- Coupled (the ecosystem is coupled to the agent)

## Future 3: Category

The future is category. The agent will define a
category. The agent will be the default.

**Examples:**
- "gitagent is the git agent"
- "gitagent is the agent framework"
- "gitagent is the agent standard"

**When to use:** the user is asking about the future.
The user wants to lead. The maintainer is the leader.

**Pros:**
- Default (the agent is the default)
- Network (the agent has network effects)
- Moat (the agent has a moat)

**Cons:**
- Slow (the category is slow to build)
- Coupled (the category is coupled to the agent)
- Risk (the category can be replaced)

## The 3 together

The 3 are the futures. The futures are the possibilities.
The possibilities are the value.

| Future | Time | Cost | Best for |
|---|---|---|---|
| Features | Short | Low | Shipping |
| Ecosystem | Medium | Medium | Community |
| Category | Long | High | Leadership |

The future that matches the ambition is the right future.

## The 80/20

80% of the value comes from:
- Features (the user wants more)
- Ecosystem (the user wants to participate)

20% comes from:
- Category (the user wants to lead)

Focus on the 80% first. Add the 20% as you grow.

## The choice

For each agent, ask:
- Does the user want more? (features)
- Does the user want to participate? (ecosystem)
- Does the user want to lead? (category)

The right answer is the right future at the right time.

## The lesson

3 futures. 1 future model. 1 lesson: design for the right
one.

The future that matches the ambition is the right future.
The future that doesn't match is the wrong future.

The agent era is here. The future is the design. The
design is the choice. The choice is the discipline.
