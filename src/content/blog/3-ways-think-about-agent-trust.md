3 ways to think about trust: capability, intent,
consistency. Each is a different trust dimension. The
framework, the examples, the lesson.

## Dimension 1: Capability

The agent is capable. The agent can do the task. The
agent is competent.

**Examples:**
- "The agent labels 95% of issues correctly"
- "The agent drafts 90% of release notes correctly"
- "The agent responds to 85% of comments helpfully"

**When to use:** the user wants accuracy. The user
trusts the agent. The agent is critical.

**Pros:**
- Trust (the user trusts the agent)
- Adoption (the user adopts the agent)
- Quality (the agent is high quality)

**Cons:**
- Cost (the capability costs money)
- Latency (the capability takes time)
- Risk (the capability can be wrong)

## Dimension 2: Intent

The agent has good intent. The agent wants the same as
the user. The agent is aligned.

**Examples:**
- "The agent labels as bug because it's a bug, not to
  game metrics"
- "The agent drafts a release note that helps the user,
  not the agent's metrics"

**When to use:** the user wants alignment. The user
trusts the agent. The agent is long-lived.

**Pros:**
- Aligned (the agent is aligned with the user)
- Trust (the user trusts the agent)
- Long-term (the agent is long-term)

**Cons:**
- Hard (the intent is hard to measure)
- Risk (the intent can drift)
- Cost (the intent takes effort)

## Dimension 3: Consistency

The agent is consistent. The agent does the same thing
the same way. The agent is reliable.

**Examples:**
- "The agent labels the same issue the same way every
  time"
- "The agent drafts the same release note every time"
- "The agent responds the same way every time"

**When to use:** the user wants predictability. The user
trusts the agent. The agent is reliable.

**Pros:**
- Predictable (the agent is predictable)
- Trust (the user trusts the agent)
- Reliable (the agent is reliable)

**Cons:**
- Cost (the consistency costs money)
- Limited (the consistency can be limited)
- Rigid (the consistency can be rigid)

## The 3 together

The 3 are the dimensions. The dimensions are the trust.
The trust is the value.

| Dimension | What it ensures |
|---|---|
| Capability | The user can rely |
| Intent | The user can trust |
| Consistency | The user can predict |

The agent that has all 3 is the trusted agent. The agent
that has 1 is the untrusted agent. The trusted agent is
adopted. The untrusted agent is not.

## The 80/20

80% of the value comes from:
- Capability (the user can rely)
- Consistency (the user can predict)

20% comes from:
- Intent (the user can trust)

Focus on the 80% first. Add the 20% as you grow.

## The choice

For each agent, ask:
- Is the agent capable? (capability)
- Is the agent aligned? (intent)
- Is the agent consistent? (consistency)

The right answer is the right dimension at the right
cost.

## The lesson

3 dimensions. 1 trust. 1 lesson: invest in all 3.

The trust that invests in all 3 is the complete trust.
The trust that invests in 1 is the partial trust. The
complete trust is adopted. The partial trust is not.

The agent era is here. The trust is the design. The
design is the discipline. The discipline is the value.
