Some developers ship 10x more than others. The 10x isn't a myth.
It's real, measurable, and reproducible. This post is what it is
and how to get there.

## The 10x isn't typing speed

A 10x developer doesn't type 10x faster. They don't read 10x
faster. They don't have 10x better memory.

The 10x is in:
- **Leverage.** Tools that multiply effort.
- **Process.** Systems that scale attention.
- **Focus.** The discipline to do the important thing first.
- **Meta-game.** The ability to play the game at multiple levels.

A 10x developer with the same tools as a 1x developer is still
10x. The tools help, but they're not the multiplier.

## The leverage

A 10x developer uses tools that 1x developers don't:
- **AI assistants.** Claude, Claude Code, Cursor. Not as
  autocomplete but as a peer reviewer.
- **Static analysis.** TypeScript strict, ESLint, Biome. Catch
  bugs at compile time.
- **Test automation.** Vitest, Playwright. Tests run on every
  push.
- **CI/CD.** GitHub Actions, Fly.io. Deploy on every merge.
- **Version control.** Git, jj. Atomic commits, easy reverts.
- **Editor.** Neovim, Helix, Zed. Keyboard-first, composable.

Each tool is a 2x-5x multiplier. The 10x developer has 5-10 of
these. The 1x developer has 1-2.

## The process

A 10x developer has a process that scales attention:
- **One task at a time.** No multitasking. The current task gets
  all the focus.
- **Small commits.** Each commit is one logical change. Easy to
  review, easy to revert.
- **Tests first.** Write the test, then make it pass. The test
  is the spec.
- **Ship often.** Daily or weekly. Each ship is a learning
  opportunity.
- **Document as you go.** The README is a byproduct, not a
  chore.

A 1x developer writes code for weeks, then "polishes" at the end.
A 10x developer ships 10 small things in the same time.

## The focus

A 10x developer has the discipline to do the important thing
first:
- **The 80/20.** 80% of the value comes from 20% of the features.
  Find the 20% and ship it.
- **The MVP.** The minimum viable product. Ship the smallest
  thing that works.
- **The daily review.** Every day, ask "what's the most important
  thing I can ship today?"
- **The kill list.** Some things are never worth doing. List them.

A 1x developer has 10 projects at 10% completion. A 10x developer
has 1 project at 100% and 9 killed.

## The meta-game

A 10x developer plays the game at multiple levels:
- **The daily level.** Ship one thing today.
- **The weekly level.** Ship one feature this week.
- **The monthly level.** Ship one milestone this month.
- **The yearly level.** Build a body of work this year.

The 1x developer thinks in days. The 10x developer thinks in
years. The 10x is compounding; the 1x is not.

## How to get there

### 1. Audit your tools

List the tools you use. For each one, ask: "is this the best tool
for the job?" If the answer is "I don't know," research. If the
answer is "no," switch.

A 10x developer has audited their tools. A 1x developer hasn't.

### 2. Audit your process

List the steps you take on a typical task. For each step, ask:
"is this necessary?" If no, delete. If yes, automate.

A 10x developer has automated the boring parts. A 1x developer
hasn't.

### 3. Audit your focus

List the projects you're working on. For each, ask: "is this the
most important thing?" If no, kill it.

A 10x developer has 1-2 active projects. A 1x developer has 10.

### 4. Audit your meta-game

Look at the last year. How many things did you ship? How many
features? How many releases? How many users?

If the answer is "I don't know," you don't have a meta-game. If
the answer is "10 features, 5 releases, 100 users," you do.

## The 10x in practice

Here's my last 30 days:
- **6 repositories** active
- **50+ commits** across them
- **10 features** shipped
- **5 releases** published
- **90 pages** of blog posts written
- **3 hours/day** of actual coding (the rest is reading, planning,
  writing)

I'm not a 10x developer. I'm a 1x-2x developer. The audit above
shows me where the gaps are.

## The lesson

The 10x isn't a myth. It's a multiplier on the underlying work.
The 10x developer has the discipline to ship 10x more in the same
time because the leverage compounds.

You can get there. It's not talent. It's tools, process, focus,
meta-game. Each is auditable. Each is improvable. The 10x is a
journey, not a destination.

Start with one audit. Pick the area where you're weakest. Spend
a week improving. Then audit again.

The compounding starts on day one.
