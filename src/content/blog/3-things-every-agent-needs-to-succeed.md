After 3 years of building and running agents, the 3 things
that determine success: a clear purpose, a tight feedback loop,
a sustainable pace. The framework, the examples, the lesson.

## Thing 1: A clear purpose

The agent exists for a reason. The reason is clear. The user
knows the reason. The agent knows the reason.

**Example:** A triage agent exists to "label new issues,
ask for repro on bugs, and close obvious duplicates." The
purpose is clear. The user knows what the agent does. The
agent knows what to do.

**Counter-example:** An agent that exists to "help with
issues." The purpose is vague. The user doesn't know what
the agent does. The agent doesn't know what to do.

The clear purpose:
- Tells the user what to expect
- Tells the agent what to do
- Tells the maintainer what to optimize
- Tells the team what to extend

The clear purpose is the foundation. Without it, the agent
is a toy. With it, the agent is a tool.

## Thing 2: A tight feedback loop

The agent gets feedback. The user reviews the agent's output.
The user tells the agent what to do differently. The agent
updates its behavior. The user reviews again. The cycle
repeats.

**Example:** A user runs the agent 10 times. Each time, the
user reviews the output. The user gives feedback. The agent
updates. The user reviews again. After 50 runs, the agent's
accuracy has improved from 60% to 95%.

**Counter-example:** A user runs the agent 10 times. The user
never reviews. The agent never improves. After 100 runs, the
agent's accuracy is still 60%.

The tight feedback loop:
- Tells the agent what's working
- Tells the agent what's not working
- Tells the agent what to change
- Compounds over time

The feedback loop is the iteration. The iteration is the
improvement. The improvement is the value.

## Thing 3: A sustainable pace

The agent runs at a pace the maintainer can sustain. Not too
fast (so the maintainer burns out). Not too slow (so the
agent becomes irrelevant).

**Example:** A maintainer who can review 10 runs per day,
5 days per week, for 50 weeks per year = 2,500 runs per year.
The maintainer can sustain this for years.

**Counter-example:** A maintainer who reviews 100 runs per day
for 2 weeks, then burns out. The maintainer stops. The agent
becomes a zombie.

The sustainable pace:
- Matches the maintainer's capacity
- Matches the user's need
- Matches the agent's value
- Compounds over time

The sustainable pace is the discipline. The discipline is the
longevity. The longevity is the value.

## The 3 together

The 3 things compose. The 3 are the foundation.

| Thing | What it ensures |
|---|---|
| Clear purpose | The agent does the right thing |
| Tight feedback loop | The agent improves over time |
| Sustainable pace | The agent lasts for years |

The agent with all 3 is durable. The agent missing any is
fragile.

## The 80/20

80% of the value comes from:
- Clear purpose (the agent does the right thing)
- Tight feedback loop (the agent improves over time)

20% comes from:
- Sustainable pace (the agent lasts for years)

Focus on the 80% first. Add the 20% as you grow.

## The test

The agent is successful if:
- A new user can describe what the agent does in 1 sentence
  (clear purpose)
- The agent's accuracy has improved over 50 runs
  (tight feedback loop)
- The maintainer has shipped a release in the last 30 days
  (sustainable pace)

If any of these fail, the agent is failing. Fix the agent.

## The meta-lesson

The 3 things are the foundation. The foundation is the
success. The success is the value.

The agent era is here. The success is the choice. The choice
is the discipline. The discipline is the foundation.

The agent that follows the 3 is durable. The agent that
doesn't is a toy. The choice is yours.
