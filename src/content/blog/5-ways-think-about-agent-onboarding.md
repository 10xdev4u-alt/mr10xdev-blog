5 ways to think about onboarding: docs, examples, templates,
video, community. Each is a different onboarding lever. The
framework, the examples, the lesson.

## Lever 1: Docs

The docs are the manual. The docs are the reference. The
docs are the truth.

**Examples:**
- README with quick start
- Manifest spec
- Tools reference
- Architecture overview

**When to use:** the user is technical. The user reads. The
user wants the truth.

**Pros:**
- Comprehensive (covers everything)
- Searchable (the user can find)
- Reference (the user can come back)

**Cons:**
- Slow (the user has to read)
- Dry (no interaction)
- Impersonal (no help)

## Lever 2: Examples

The examples are the templates. The examples are the
starting points. The examples are the truth in action.

**Examples:**
- Triage agent manifest
- Doc agent manifest
- Release agent manifest
- Code review agent manifest

**When to use:** the user wants to copy. The user wants to
modify. The user wants to learn by example.

**Pros:**
- Fast (the user copies)
- Real (the user sees real code)
- Educational (the user learns)

**Cons:**
- Limited (the user is constrained)
- Outdated (the user might copy old)
- Incomplete (the user has to fill in)

## Lever 3: Templates

The templates are the scaffolds. The templates are the
starting points. The templates are the structure.

**Examples:**
- `gitagent init` (scaffolds a new repo)
- `gitagent new <name>` (scaffolds a new agent)
- `gitagent validate` (validates the config)

**When to use:** the user wants to start fast. The user
doesn't know what to do. The user needs a guide.

**Pros:**
- Fastest (the user runs a command)
- Guided (the user is led)
- Validated (the user knows it works)

**Cons:**
- Rigid (the user is constrained)
- Opaque (the user doesn't learn)
- Limited (the templates cover the common cases)

## Lever 4: Video

The video is the demo. The video is the tutorial. The video
is the visual.

**Examples:**
- "Build an agent in 5 minutes" (YouTube)
- "Triage agent walkthrough" (Loom)
- "Deploy an agent to GitHub Actions" (YouTube)

**When to use:** the user is visual. The user is in a hurry.
The user wants to see.

**Pros:**
- Visual (the user sees)
- Fast (the user watches in 5 minutes)
- Engaging (the user is hooked)

**Cons:**
- Passive (the user doesn't do)
- Outdated (the video goes stale)
- Limited (the video is one path)

## Lever 5: Community

The community is the help. The community is the support.
The community is the network.

**Examples:**
- Discord channel
- GitHub Discussions
- Office hours
- Office AMAs

**When to use:** the user is stuck. The user has a question.
The user wants to learn from others.

**Pros:**
- Helpful (the user gets a real answer)
- Networked (the user meets others)
- Living (the user gets the latest)

**Cons:**
- Slow (the user has to wait)
- Inconsistent (the user gets different answers)
- Dependent (the user is dependent on the community)

## The 5 levers together

The 5 are the onboarding. The onboarding is the adoption.
The adoption is the growth.

| Lever | Speed | Depth | Best for |
|---|---|---|---|
| Docs | Slow | Deep | Reference |
| Examples | Fast | Medium | Learning |
| Templates | Fastest | Shallow | Starting |
| Video | Fast | Medium | Visual |
| Community | Slow | Deep | Stuck |

The agent that uses all 5 is the agent that onboards. The
agent that uses 1 is the agent that loses users.

## The 80/20

80% of the value comes from:
- Docs (the user has the truth)
- Examples (the user has the templates)

20% comes from:
- Templates (the user has the scaffold)
- Video (the user has the demo)
- Community (the user has the help)

Focus on the 80% first. Add the 20% as you grow.

## The order

The right order:
1. **README** (the user lands on the repo)
2. **Quick start** (the user runs the example in 5 minutes)
3. **Examples** (the user copies an example)
4. **Docs** (the user reads when stuck)
5. **Community** (the user asks when stuck)

The order is the journey. The journey is the adoption.

## The lesson

5 levers. 1 onboarding. 1 lesson: invest in all 5.

The agent that invests in all 5 is the onboarded agent. The
agent that invests in 1 is the lost agent. The onboarded
agent is adopted. The lost agent is not.

The agent era is here. The onboarding is the design. The
design is the discipline. The discipline is the adoption.
