100 agents later, here are the 5 things I wish I'd known.
The framework, the examples, the lesson. The hard truths
from the trenches.

## Lesson 1: The hard part isn't the agent

The hard part is the integration. The agent is the easy
part. The hard part is connecting the agent to the systems
the user already uses.

**What I expected:** "I'll build a smart agent that triages
issues!"

**What it was:** "I'll build a smart agent, then connect it
to GitHub, then handle auth, then handle webhooks, then
handle retries, then handle rate limits, then handle
timeouts, then handle..."

The agent is 20% of the work. The integration is 80% of the
work. The lesson: budget for the integration.

## Lesson 2: The user doesn't care about your framework

The user cares about: "does it work?" The user doesn't care
about: "is it a state machine?" The user doesn't care about:
"is it event-driven?"

**What I expected:** "The user will love my elegant
framework!"

**What it was:** "The user wants the issue triaged. The
user doesn't care how."

The framework is for you. The user cares about the
outcome. The lesson: design for the outcome, not the
framework.

## Lesson 3: The first version will be wrong

The first version will be wrong. The second version will
be less wrong. The third version will be almost right. The
fourth version will be right.

**What I expected:** "I'll get it right the first time!"

**What it was:** "I'll get it wrong the first 3 times."

The first version is a learning exercise. The lesson: ship
the first version, learn from the user, ship the second
version.

## Lesson 4: The 80/20 is the prompt

80% of the agent's quality comes from the prompt. 20% comes
from the framework.

**What I expected:** "The framework is the value."

**What it was:** "The prompt is the value."

The framework is the plumbing. The prompt is the soul. The
lesson: invest in the prompt. The prompt is the product.

## Lesson 5: The user is the teacher

The user knows more than you do. The user knows the domain.
The user knows the use case. The user knows what works.

**What I expected:** "I'll teach the user how to use the
agent."

**What it was:** "The user will teach me how to build the
agent."

The user is the teacher. The agent builder is the student.
The lesson: listen to the user. The user is the truth.

## The 5 lessons together

The 5 are the truth. The truth is the discipline. The
discipline is the agent.

| Lesson | What it taught me |
|---|---|
| The hard part isn't the agent | Budget for integration |
| The user doesn't care about your framework | Design for the outcome |
| The first version will be wrong | Ship and learn |
| The 80/20 is the prompt | Invest in the prompt |
| The user is the teacher | Listen to the user |

The lessons are the hard truths. The hard truths are the
lessons. The lessons are the discipline.

## The 80/20

80% of the value comes from:
- Lesson 1 (the integration is the work)
- Lesson 3 (the first version will be wrong)

20% comes from:
- Lesson 2 (the user cares about the outcome)
- Lesson 4 (the prompt is the product)
- Lesson 5 (the user is the teacher)

Focus on the 80% first. Add the 20% as you grow.

## The lesson

5 lessons. 1 career. 1 lesson: the user is the truth.

The agent builder that listens to the user is the agent
builder that succeeds. The agent builder that doesn't is
the agent builder that fails.

The agent era is here. The truth is the design. The design
is the discipline. The discipline is the success.
