4 ways to make an agent feel useful: relevant, actionable,
accurate, timely. The principles that make agents valuable.
The line between "useless" and "useful."

## Principle 1: Relevant

The agent does what the user needs. The agent understands
the context. The agent fits the workflow.

**Bad:** the agent labels every issue as "bug." The agent
is generic. The user has to fix it.

**Good:** the agent labels the issue based on the context.
The agent is specific. The user is happy.

The relevant is the difference between "the agent is
generic" and "the agent is specific."

## Principle 2: Actionable

The agent's output is something the user can do. The
agent's output is concrete. The agent's output is useful.

**Bad:** the agent says "this might be a bug." The user
doesn't know what to do.

**Good:** the agent says "this is a bug in src/auth.ts at
line 42. Here's the fix: ..."

The actionable is the difference between "the agent
vaguely knows" and "the agent concretely knows."

## Principle 3: Accurate

The agent is right. The agent is correct. The agent
doesn't make mistakes. The agent is reliable.

**Bad:** the agent is right 50% of the time. The user
doesn't trust. The user double-checks.

**Good:** the agent is right 95% of the time. The user
trusts. The user delegates.

The accurate is the difference between "let me check" and
"I'll trust it."

## Principle 4: Timely

The agent is fast. The agent is on time. The agent
delivers when the user needs it.

**Bad:** the agent takes 30 seconds. The user has already
moved on.

**Good:** the agent takes 1 second. The user is hooked.

The timely is the difference between "too late" and
"just in time."

## The 4 together

The 4 are the useful. The useful is the value. The value
is the adoption.

| Principle | What it ensures |
|---|---|
| Relevant | The user is interested |
| Actionable | The user can act |
| Accurate | The user can trust |
| Timely | The user can use |

The agent that has all 4 is the useful agent. The agent
that has 1 is the useless agent. The useful agent is
adopted. The useless agent is not.

## The 80/20

80% of the value comes from:
- Accurate (the user trusts)
- Actionable (the user acts)

20% comes from:
- Relevant (the user cares)
- Timely (the user uses)

Focus on the 80% first. Add the 20% as you grow.

## The test

The agent feels useful if:
- A user can act on the output
- A user can trust the output
- A user cares about the output
- A user gets the output in time

If any of these fails, the agent feels useless. Fix the
agent.

## The lesson

4 principles. 1 useful. 1 lesson: design for the user.

The agent that designs for the user is the useful agent.
The agent that designs for itself is the useless agent.
The useful agent is adopted. The useless agent is not.

The agent era is here. The useful is the design. The
design is the discipline. The discipline is the value.
