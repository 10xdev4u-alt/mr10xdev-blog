4 ways to make an agent feel personal: named, contextual,
history, learning. The principles that make agents feel
personal. The line between "generic" and "personal."

## Principle 1: Named

The agent has a name. The agent is "Alice the Triager."
The agent is a person, not a process.

**Bad:** the agent is "triage-bot." The agent is a
process. The agent is generic.

**Good:** the agent is "Alice the Triager." The agent is
a person. The agent is personal.

The named is the difference between "the agent is a
process" and "the agent is a person."

## Principle 2: Contextual

The agent is contextual. The agent knows the user. The
agent knows the project. The agent knows the workflow.

**Bad:** the agent is generic. The agent treats the user
like everyone else. The agent is impersonal.

**Good:** the agent knows the user. The agent says
"Welcome back, Alice. I see you have 3 new issues." The
agent is personal.

The contextual is the difference between "the agent is
generic" and "the agent is personal."

## Principle 3: History

The agent has history. The agent remembers. The agent
knows the past.

**Bad:** the agent forgets. The agent treats each
interaction as new. The agent is impersonal.

**Good:** the agent remembers. The agent says "Last time
you asked about issue #42. I closed it as duplicate of
#41." The agent is personal.

The history is the difference between "the agent is
amnesia" and "the agent is personal."

## Principle 4: Learning

The agent learns. The agent improves. The agent grows
with the user.

**Bad:** the agent is static. The agent doesn't learn.
The agent is impersonal.

**Good:** the agent learns. The agent says "I've noticed
you prefer short responses. I'll keep that in mind." The
agent is personal.

The learning is the difference between "the agent is
static" and "the agent is personal."

## The 4 together

The 4 are the personal. The personal is the warmth. The
warmth is the value.

| Principle | What it ensures |
|---|---|
| Named | The agent is a person |
| Contextual | The agent is for the user |
| History | The agent is with the user |
| Learning | The agent is for the user |

The agent that has all 4 is the personal agent. The
agent that has 1 is the generic agent. The personal agent
is adopted. The generic agent is not.

## The 80/20

80% of the value comes from:
- Named (the agent is a person)
- Contextual (the agent is for the user)

20% comes from:
- History (the agent is with the user)
- Learning (the agent is for the user)

Focus on the 80% first. Add the 20% as you grow.

## The test

The agent feels personal if:
- A user feels the agent is a person
- A user feels the agent is for them
- A user feels the agent is with them
- A user feels the agent is growing with them

If any of these fails, the agent feels generic. Fix the
agent.

## The lesson

4 principles. 1 personal. 1 lesson: invest in the
principles.

The agent that invests in the principles is the personal
agent. The agent that doesn't is the generic agent. The
personal agent is adopted. The generic agent is not.

The agent era is here. The personal is the design. The
design is the discipline. The discipline is the value.
