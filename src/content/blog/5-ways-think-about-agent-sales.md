5 ways to think about sales: free, freemium, paid,
enterprise, custom. Each is a different sales model. The
framework, the examples, the lesson.

## Model 1: Free

The agent is free. The agent has no price. The agent is
the truth.

**Examples:**
- "Free for everyone"
- "Free for public repos"
- "Free for personal use"

**When to use:** the user is open source. The user is
adoption-focused. The user is large.

**Pros:**
- Adoption (the user is widely adopted)
- Community (the user builds a community)
- Word-of-mouth (the user spreads the word)

**Cons:**
- No revenue (the user has no revenue)
- Cost (the user pays for the infra)
- Limit (the user can't scale without revenue)

## Model 2: Freemium

The agent is free for some. The agent is paid for others.
The agent is the truth.

**Examples:**
- "Free for 100 runs/month"
- "Paid for unlimited runs"
- "Free for personal use, paid for commercial use"

**When to use:** the user has a clear free tier. The user
has a clear paid tier. The user is hybrid.

**Pros:**
- Adoption (the user is widely adopted)
- Revenue (the user has revenue)
- Conversion (the user can convert)

**Cons:**
- Complexity (the user has two tiers)
- Support (the user supports both tiers)
- Abuse (the user can be abused)

## Model 3: Paid

The agent is paid. The agent has a price. The agent is
the truth.

**Examples:**
- "$9/month per user"
- "$99/month per team"
- "$999/month per org"

**When to use:** the user is a business. The user wants
quality. The user is committed.

**Pros:**
- Revenue (the user has clear revenue)
- Quality (the user can invest in quality)
- Sustainability (the user is sustainable)

**Cons:**
- Friction (the user has friction)
- Conversion (the user has to convert)
- Limit (the user has fewer users)

## Model 4: Enterprise

The agent is enterprise. The agent is custom-priced. The
agent is the truth.

**Examples:**
- "Contact us for pricing"
- "Custom contract per customer"
- "SLA-backed"

**When to use:** the user is enterprise. The user wants
custom. The user is large.

**Pros:**
- Revenue (the user has high revenue)
- Custom (the user is custom to the customer)
- Long-term (the user is long-term)

**Cons:**
- Sales (the user needs sales)
- Complexity (the user has complex sales)
- Slow (the user is slow to close)

## Model 5: Custom

The agent is custom. The agent is bespoke. The agent is
the truth.

**Examples:**
- "We'll build the agent for you"
- "We'll customize the agent for you"
- "We'll train the agent on your data"

**When to use:** the user has unique needs. The user is
willing to pay. The user is committed.

**Pros:**
- High value (the user has high value)
- Custom (the user is fully custom)
- Long-term (the user is long-term)

**Cons:**
- Effort (the user takes effort)
- Coupling (the user is coupled to the customer)
- Limit (the user can't scale)

## The 5 together

The 5 are the models. The models are the sales. The
sales are the revenue.

| Model | Adoption | Revenue | Best for |
|---|---|---|---|
| Free | High | None | OSS |
| Freemium | High | Medium | Hybrid |
| Paid | Medium | High | Business |
| Enterprise | Low | Highest | Large |
| Custom | Lowest | High | Unique |

The model that matches the agent is the right model.

## The 80/20

80% of the value comes from:
- Free (the user is adopted)
- Freemium (the user is hybrid)

20% comes from:
- Paid (the user is business)
- Enterprise (the user is large)
- Custom (the user is unique)

Focus on the 80% first. Add the 20% as you grow.

## The choice

For each agent, ask:
- Is the agent open source? (free)
- Is the agent hybrid? (freemium)
- Is the agent business? (paid)
- Is the agent enterprise? (enterprise)
- Is the agent unique? (custom)

The right answer is the right model at the right scale.

## The lesson

5 models. 1 sales. 1 lesson: pick the right one.

The model that matches the agent is the right model. The
model that doesn't match is the wrong model.

The agent era is here. The sales is the design. The
design is the choice. The choice is the discipline.
