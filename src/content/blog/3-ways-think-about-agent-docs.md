3 ways to think about docs: README, reference, guide. Each
is a different docs type. The framework, the examples, the
lesson.

## Docs 1: README

The README is the first impression. The README is the
quick start. The README is the truth.

**Examples:**
- "What is this?"
- "How do I install it?"
- "How do I use it?"

**When to use:** the user is new. The user is in a hurry.
The user wants the basics.

**Pros:**
- First (the README is the first thing)
- Fast (the user can read in 5 minutes)
- Standard (the README is the convention)

**Cons:**
- Shallow (the README is shallow)
- Static (the README can be outdated)
- Limited (the README can't cover everything)

## Docs 2: Reference

The reference is the manual. The reference is the API.
The reference is the truth.

**Examples:**
- "What tools are available?"
- "What are the input/output of each tool?"
- "What are the limits?"

**When to use:** the user is experienced. The user is
looking up. The user wants the details.

**Pros:**
- Comprehensive (the reference covers everything)
- Searchable (the user can search)
- Accurate (the reference is auto-generated)

**Cons:**
- Dry (the reference is dry)
- Boring (the reference is boring)
- Coupled (the reference is tied to the API)

## Docs 3: Guide

The guide is the tutorial. The guide is the how-to. The
guide is the truth.

**Examples:**
- "How to build a triage agent"
- "How to deploy an agent to GitHub Actions"
- "How to test an agent"

**When to use:** the user is learning. The user is doing
a task. The user wants the steps.

**Pros:**
- Practical (the guide is practical)
- Step-by-step (the guide is step-by-step)
- Engaging (the guide is engaging)

**Cons:**
- Specific (the guide is specific to one path)
- Outdated (the guide can be outdated)
- Effort (the guide takes effort to write)

## The 3 together

The 3 are the docs. The docs are the truth. The truth is
the value.

| Docs | Audience | Depth |
|---|---|---|
| README | New user | Shallow |
| Reference | Experienced user | Deep |
| Guide | Learning user | Practical |

The docs that match the audience is the right docs.

## The 80/20

80% of the value comes from:
- README (the user can start)
- Guide (the user can learn)

20% comes from:
- Reference (the user can look up)

Focus on the 80% first. Add the 20% as you grow.

## The choice

For each agent, ask:
- Is the user new? (README)
- Is the user learning? (Guide)
- Is the user experienced? (Reference)

The right answer is the right docs at the right depth.

## The lesson

3 docs. 1 documentation. 1 lesson: write all 3.

The docs that write all 3 is the complete docs. The docs
that write 1 is the partial docs. The complete docs is
adopted. The partial docs is not.

The agent era is here. The docs are the design. The
design is the discipline. The discipline is the value.
