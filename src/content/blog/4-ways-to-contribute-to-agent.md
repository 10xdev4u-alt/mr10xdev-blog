4 ways to contribute: code, docs, examples, community. Each
has different value, different effort, different recognition.
The framework for finding your way in.

## Way 1: Code

The most direct contribution. The contributor writes code
that improves the project.

**What the contribution looks like:**
- A bug fix
- A new feature
- A refactor
- A test
- A performance improvement

**The value:** the project gets better. The bug is fixed.
The feature is added. The test catches regressions.

**The effort:** medium to high. Code requires understanding
the codebase, the conventions, the design.

**The recognition:** the contributor's name is in the git
log. The contributor is in the release notes. The
contributor is in the README contributors list.

## Way 2: Docs

The docs are the contract with the user. The contributor
writes docs that explain the project.

**What the contribution looks like:**
- A new doc page
- An example in the docs
- A typo fix
- A clarification
- A tutorial

**The value:** the project is more usable. The user
understands the project. The user can adopt the project.

**The effort:** low to medium. Docs require understanding the
project, the audience, the writing style.

**The recognition:** the contributor's name is in the git
log. The contributor might be in the contributors list.

## Way 3: Examples

Examples are the most concrete contribution. The contributor
writes an example that shows how to use the project.

**What the contribution looks like:**
- A new example
- A real-world use case
- A recipe (combining multiple features)
- A showcase (a notable user of the project)

**The value:** the project is more approachable. The user
sees a working example. The user can adapt the example.

**The effort:** medium. Examples require understanding the
project, the use case, the code.

**The recognition:** the contributor's name is in the git
log. The example is attributed. The contributor is featured
in the README.

## Way 4: Community

The community is the moat. The contributor helps the
community grow.

**What the contribution looks like:**
- Answering questions in issues
- Helping users in Discord
- Reviewing PRs
- Triaging issues
- Writing blog posts or tutorials
- Speaking at conferences

**The value:** the project is more sustainable. The
community grows. The contributors find each other. The
maintainer is less stressed.

**The effort:** low to high. Community is whatever you can
give.

**The recognition:** the contributor is in the contributors
list. The contributor is featured in the README. The
contributor is invited to be a co-maintainer.

## The 4 together

The 4 ways are not exclusive. The best contributors do all
4. They write code, they write docs, they write examples, they
help the community.

| Way | Value | Effort | Recognition |
|---|---|---|---|
| Code | High | Med-high | Git log, release notes |
| Docs | Med | Low-med | Git log, contributors list |
| Examples | High | Med | Git log, README, featured |
| Community | High | Low-high | Git log, contributors list, co-maintainer |

The 4 ways are the on-ramps. The on-ramps are the entry
points. The entry points are the participation.

## The 80/20

80% of the value comes from:
- Code (the project gets better)
- Community (the project is sustainable)

20% comes from:
- Docs (the project is more usable)
- Examples (the project is more approachable)

The 80% is the technical contribution. The 20% is the
documentation contribution.

The 80% is what the maintainer values. The 20% is what the
user values. Both are needed.

## The first contribution

If you're new to a project, the first contribution is the
hardest. The first contribution is the cold start.

**The fix:** start small. Fix a typo. Add an example. Answer
a question. The first contribution is a foot in the door.
The foot in the door is the start of the relationship.

After the first contribution, the second is easier. After
the second, the third is easier. The compounding is real.

## The lesson

4 ways. 1 contribution. 1 lesson: pick the way that fits you.

The contributor who codes contributes code. The contributor
who writes contributes docs. The contributor who talks
contributes community. The contributor who shows contributes
examples.

The agent era is here. The contributions are the same. The
on-ramps are the same. The choice is yours.
