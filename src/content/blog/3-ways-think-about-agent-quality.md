3 ways to think about quality: correctness, completeness,
consistency. Each is a different quality dimension. The
framework, the examples, the lesson.

## Dimension 1: Correctness

The agent is correct. The agent is right. The agent
doesn't make mistakes.

**Examples:**
- "The agent labels 95% of issues correctly"
- "The agent drafts 90% of release notes correctly"
- "The agent responds to 85% of comments helpfully"

**When to use:** the user wants accuracy. The user trusts
the agent. The agent is critical.

**Pros:**
- Trust (the user trusts the agent)
- Adoption (the user adopts the agent)
- Quality (the agent is high quality)

**Cons:**
- Cost (the correctness costs money)
- Latency (the correctness takes time)
- Risk (the correctness can be wrong)

## Dimension 2: Completeness

The agent is complete. The agent handles all cases. The
agent doesn't miss anything.

**Examples:**
- "The agent handles 100% of issue types"
- "The agent handles 100% of PR types"
- "The agent handles 100% of comment types"

**When to use:** the user wants comprehensive. The user
trusts the agent. The agent is general.

**Pros:**
- Comprehensive (the agent handles all)
- Trust (the user trusts the agent)
- General (the agent is general)

**Cons:**
- Cost (the completeness costs money)
- Complexity (the completeness is complex)
- Risk (the completeness can miss)

## Dimension 3: Consistency

The agent is consistent. The agent is the same every
time. The agent is predictable.

**Examples:**
- "The agent labels the same issue the same way every
  time"
- "The agent drafts the same release note every time"
- "The agent responds the same way every time"

**When to use:** the user wants predictability. The user
trusts the agent. The agent is reliable.

**Pros:**
- Predictable (the agent is predictable)
- Trust (the user trusts the agent)
- Reliable (the agent is reliable)

**Cons:**
- Cost (the consistency costs money)
- Limited (the consistency can be limited)
- Rigid (the consistency can be rigid)

## The 3 together

The 3 are the dimensions. The dimensions are the
quality. The quality is the value.

| Dimension | What it ensures |
|---|---|
| Correctness | The user can trust |
| Completeness | The user is covered |
| Consistency | The user can rely |

The agent that has all 3 is the quality agent. The agent
that has 1 is the mediocre agent. The quality agent is
adopted. The mediocre agent is not.

## The 80/20

80% of the value comes from:
- Correctness (the user trusts)
- Consistency (the user relies)

20% comes from:
- Completeness (the user is covered)

Focus on the 80% first. Add the 20% as you grow.

## The choice

For each agent, ask:
- Is the agent accurate? (correctness)
- Is the agent comprehensive? (completeness)
- Is the agent consistent? (consistency)

The right answer is the right dimension at the right
cost.

## The lesson

3 dimensions. 1 quality. 1 lesson: invest in all 3.

The agent that invests in all 3 is the quality agent. The
agent that invests in 1 is the mediocre agent. The quality
agent is adopted. The mediocre agent is not.

The agent era is here. The quality is the design. The
design is the discipline. The discipline is the value.
