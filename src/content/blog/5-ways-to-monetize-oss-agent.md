Sponsorship, dual-license, hosted, services, support. The 5
models for making money from an open-source agent. The
tradeoffs, the math, when to use which.

## Model 1: Sponsorship

The user pays to support the project. No feature gate. No
license. Just money to support the maintainer.

**Examples:**
- GitHub Sponsors
- Open Collective
- Patreon
- Polar

**Pros:**
- No friction (the user just pays)
- Aligned incentives (the user wants you to succeed)
- No support cost (the user uses the OSS, you don't help)

**Cons:**
- Volatile (the user can stop paying)
- Low amount (most users pay $0)
- The "value" is unclear (what does the user get?)

**The math:**
- 1,000 users × 1% sponsorship rate × $10/month = $100/month
- 10,000 users × 1% sponsorship rate × $10/month = $1K/month
- 100,000 users × 1% sponsorship rate × $10/month = $10K/month

**When to use:** the project is small, the maintainer has
followers, the project is well-loved.

## Model 2: Dual-license

The project is OSS under a permissive license (MIT, Apache).
A commercial license is available for users who want to use
it without the OSS obligations.

**Examples:**
- MongoDB (before the relicensing)
- Elastic (before the relicensing)
- Some FSL-licensed projects

**Pros:**
- Aligns with the user's needs (the user picks the right
  license)
- High value (the user gets commercial support)
- Compatible with OSS (the OSS is still OSS)

**Cons:**
- Polarizing (some users hate dual-licensing)
- Legal complexity (the license is a contract)
- Low conversion (most users pick the OSS)

**The math:**
- 100 paying customers × $10K/year = $1M/year
- 1,000 paying customers × $10K/year = $10M/year

**When to use:** the project is used in commercial products,
the user needs commercial support, the license is enforceable.

## Model 3: Hosted

The OSS is free. The hosted version is paid. The user pays for
the convenience of not hosting.

**Examples:**
- GitLab
- Sentry
- Supabase
- Hasura
- Various agent platforms

**Pros:**
- Aligned with the user's needs (some users don't want to
  host)
- High value (the user gets a managed service)
- Compatible with OSS (the OSS is still OSS)

**Cons:**
- Operational cost (you run the service)
- Support cost (the user expects support)
- Churn risk (the user can leave)

**The math:**
- 100 hosted users × $50/month = $5K MRR
- 1,000 hosted users × $50/month = $50K MRR
- 10,000 hosted users × $50/month = $500K MRR

**When to use:** the user doesn't want to host, the value of
hosting is high, the operational cost is manageable.

## Model 4: Services

You sell your time to help users adopt the OSS.

**Examples:**
- Consulting agencies
- Custom development shops
- Independent consultants

**Pros:**
- Immediate revenue
- High hourly rate ($200-500/hour)
- Aligned with the user's needs (the user needs help)

**Cons:**
- Time-bound (you can't scale without more people)
- Not productized (each project is custom)
- High variance (revenue depends on sales)

**The math:**
- 20 hours/week × $300/hour × 4 weeks = $24K/month
- 40 hours/week × $300/hour × 4 weeks = $48K/month

**When to use:** the project is complex, the user needs
expertise, you have domain knowledge.

## Model 5: Support

You sell a support contract. The user pays for help when
they need it.

**Examples:**
- Various OSS companies
- WordPress support contracts
- Enterprise OSS vendors

**Pros:**
- Recurring revenue (the user pays monthly/annually)
- Aligned with the user's needs (the user wants help)
- High value (the user pays for expertise)

**Cons:**
- Sales cycle (the user has to commit to a contract)
- Support cost (the user expects responsive support)
- Churn risk (the user can cancel)

**The math:**
- 10 support contracts × $5K/year = $50K/year
- 100 support contracts × $5K/year = $500K/year
- 1,000 support contracts × $5K/year = $5M/year

**When to use:** the project is complex, the user wants
guaranteed support, you have the capacity.

## The 5 together

The 5 models are not exclusive. Most successful OSS projects
use 2-3:

- **Sponsorship + Services:** the OSS is free, the services
  are paid. The sponsorship is the marketing.
- **Hosted + Support:** the hosted is paid, the support is
  paid. The OSS is the marketing.
- **Dual-license + Sponsorship:** the OSS is free, the
  commercial license is paid. The sponsorship is the community
  signal.

The right combination depends on the project. The right
combination is what makes the business work.

## The 80/20

80% of OSS projects use sponsorship + services. The
sponsorship is the marketing. The services are the bootstrap.

20% use hosted or dual-license. The hosted is the product.
The dual-license is the enterprise.

Start with sponsorship + services. Add hosted or
dual-license as you grow.

## The lesson

5 models. 1 choice. 1 lesson: pick the model that fits.

The OSS is the marketing. The monetization is the product.
The product is the choice. The choice is the design.

The agent that has a good business model is sustainable. The
agent that has a bad business model is a hobby. The choice
is yours.
