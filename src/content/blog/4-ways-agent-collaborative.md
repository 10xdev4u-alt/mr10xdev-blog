4 ways to make an agent feel collaborative: 2-way, side-by-side,
pair-programming, code-review. Each is a different collaboration
pattern with different tradeoffs. The patterns, the examples, the
lesson.

## Pattern 1: 2-way chat

The user asks. The agent answers. The user asks again. The
agent answers again.

**Examples:**
- ChatGPT
- Claude
- Most LLMs

**Pros:**
- Simple. The user knows what to do.
- Familiar. The user has used chat before.
- Flexible. The user can ask anything.

**Cons:**
- The user is alone. The user feels alone.
- The agent doesn't know the context. The agent is generic.
- The output is text. The output is verbose.

## Pattern 2: Side-by-side

The user types. The agent suggests. The user accepts or
rejects. The user keeps typing.

**Examples:**
- GitHub Copilot
- Cursor
- Codeium

**Pros:**
- The user is in control. The user accepts/rejects.
- The agent is fast. The agent suggests in milliseconds.
- The agent is contextual. The agent suggests based on the
  code.

**Cons:**
- The agent can be distracting. The agent suggests often.
- The agent can be wrong. The agent is often wrong.
- The user can over-rely. The user stops thinking.

## Pattern 3: Pair programming

The user and the agent share an editor. The user and the
agent take turns. The user and the agent build together.

**Examples:**
- Cursor's "Composer" mode
- Replit's "Ghostwriter"
- Aider

**Pros:**
- The user and the agent are equal. The user is in the loop.
- The agent can do more. The agent can run code, search,
  refactor.
- The collaboration is rich. The collaboration is real-time.

**Cons:**
- The agent can be wrong. The agent is often wrong.
- The user can be slow. The user has to review.
- The collaboration can be confusing. The user and the
  agent have different contexts.

## Pattern 4: Code review

The user opens a PR. The agent reviews. The agent comments.
The user responds. The user and the agent iterate.

**Examples:**
- GitHub Copilot PR reviews
- Coderabbit
- Codeball

**Pros:**
- The agent is asynchronous. The agent works in the
  background.
- The agent is focused. The agent reviews one PR at a time.
- The agent is consistent. The agent applies the same rules
  every time.

**Cons:**
- The agent is late. The agent reviews after the user is
  done.
- The agent can be annoying. The agent comments on
  everything.
- The agent can be wrong. The agent doesn't know the
  context.

## The 4 together

The 4 are the collaboration. The collaboration is the
relationship. The relationship is the value.

| Pattern | Synchronicity | Control | When to use |
|---|---|---|---|
| 2-way chat | Sync | User asks, agent answers | Exploration |
| Side-by-side | Sync | User types, agent suggests | Writing |
| Pair programming | Sync | User + agent build | Building |
| Code review | Async | User writes, agent reviews | Shipping |

The agent that uses the right pattern is the agent that
matches the user's workflow. The agent that uses the wrong
pattern is the agent that frustrates the user.

## The 80/20

80% of the value comes from:
- 2-way chat (the user can ask)
- Side-by-side (the user can write)

20% comes from:
- Pair programming (the user can build)
- Code review (the user can ship)

Focus on the 80% first. Add the 20% as you grow.

## The lesson

4 patterns. 1 collaboration. 1 lesson: pick the right
pattern.

The agent that picks the right pattern is the agent that
matches the user. The agent that picks the wrong pattern
is the agent that fights the user.

The agent era is here. The collaboration is the design.
The design is the choice. The choice is the discipline.
