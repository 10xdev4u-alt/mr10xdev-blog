The 5 things agent maintainers do that kill their projects.
The abandoner, the perfectionist, the over-committer, the
gatekeeper, the lone wolf. The framework, the fixes, the
lesson.

## Anti-pattern 1: The abandoner

The maintainer starts strong. Ships a v0.1. Then disappears.
The repo has 0 commits in 6 months. The issues pile up. The
PRs go unreviewed. The project is dead.

**The cost:** the project dies. The users move to
alternatives. The reputation is damaged.

**The fix:** set a sustainable pace. Ship a release every 2-4
weeks. Even if it's just a bug fix. Even if it's just a
documentation update. The release cadence is the heartbeat.
The heartbeat keeps the project alive.

## Anti-pattern 2: The perfectionist

The maintainer ships nothing. Every feature needs to be
perfect. Every PR needs to be polished. Every release needs
to be flawless. The project is "almost ready" forever.

**The cost:** the project never ships. The users give up. The
maintainer burns out.

**The fix:** ship the smallest thing. The smallest thing is
better than the perfect thing. The user wants the working
thing, not the perfect thing.

## Anti-pattern 3: The over-committer

The maintainer says yes to everything. Every PR is merged.
Every issue is closed. Every feature is added. The project
becomes a kitchen sink. The project loses focus.

**The cost:** the project becomes unfocused. The user can't
tell what the project does. The maintainer is overwhelmed.

**The fix:** say no. Most feature requests don't fit. Most
PRs don't need to be merged. The maintainer's job is to say
no. The no is the value.

## Anti-pattern 4: The gatekeeper

The maintainer reviews every PR. The maintainer merges every
PR. The maintainer is the only one who can push. The project
depends on the maintainer.

**The cost:** the project doesn't scale. The maintainer
burns out. The community is small.

**The fix:** add co-maintainers. Trust the contributors. Let
them merge their own PRs (after review). Delegate. The
delegation is the scale.

## Anti-pattern 5: The lone wolf

The maintainer works alone. No co-maintainers. No community
managers. No issue triagers. No PR reviewers. The project
is one person.

**The cost:** the project depends on one person. The project
dies when the person stops. The project is fragile.

**The fix:** build a community. Recruit co-maintainers. Add
triage bots. Add review bots. The community is the moat. The
moat is the durability.

## The 5 together

The 5 anti-patterns are the failure modes. The failure modes
are the lessons.

| Anti-pattern | Symptom | Fix |
|---|---|---|
| Abandoner | No commits in months | Set a sustainable cadence |
| Perfectionist | Never ships | Ship the smallest thing |
| Over-committer | Kitchen sink | Say no |
| Gatekeeper | Everything depends on you | Add co-maintainers |
| Lone wolf | One person project | Build a community |

The maintainer that avoids the 5 is the maintainer that
succeeds. The maintainer that falls into 1 is the maintainer
that fails.

## The 80/20

80% of the value comes from:
- The abandoner fix (set a cadence)
- The perfectionist fix (ship the smallest thing)
- The gatekeeper fix (add co-maintainers)

20% comes from:
- The over-committer fix (say no)
- The lone wolf fix (build a community)

Focus on the 80% first. Add the 20% as you grow.

## The lesson

5 anti-patterns. 1 lesson: avoid them.

The maintainer that avoids the 5 is the maintainer that
succeeds. The maintainer that falls into 1 is the maintainer
that fails.

The agent era is here. The maintainers are the future. The
future is the discipline. The discipline is the success.
