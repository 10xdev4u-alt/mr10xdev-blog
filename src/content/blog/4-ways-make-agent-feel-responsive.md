4 ways to make an agent feel responsive: streaming,
progress, immediate, anticipating. The small touches that
signal speed. The line between "slow" and "fast."

## Way 1: Streaming

The agent shows the response as it's generated. The user
sees the first word in < 1 second. The user is hooked.

**Bad:** the agent is silent for 10 seconds. The user
thinks the agent is broken. The user gives up.

**Good:** the agent streams the response. The user sees the
first word in 1 second. The user is hooked.

The streaming is the difference between "is it broken?"
and "it's already working."

## Way 2: Progress

The agent shows what it's doing. The user sees the
progress. The user knows the agent is alive.

**Bad:** the agent is silent. The user doesn't know what's
happening. The user is anxious.

**Good:** the agent shows "thinking...", "reading file...",
"calling API...". The user knows the agent is working.

The progress is the difference between "is it stuck?" and
"it's working on it."

## Way 3: Immediate

The agent responds in < 100ms. The user doesn't wait. The
user is delighted.

**Bad:** the agent takes 5 seconds. The user waits. The
user is bored.

**Good:** the agent takes 100ms. The user is delighted.
The user is hooked.

The immediate is the difference between "ugh, the agent"
and "wow, the agent."

## Way 4: Anticipating

The agent knows what the user wants. The agent does it
before the user asks. The user is delighted.

**Bad:** the user has to ask. The user has to specify. The
user has to wait.

**Good:** the agent knows the user. The agent anticipates
the need. The agent does it.

The anticipating is the difference between "I have to ask"
and "the agent already did it."

## The 4 together

The 4 are the responsive. The responsive is the speed. The
speed is the value.

| Way | What it ensures | Cost |
|---|---|---|
| Streaming | The user sees the first word | Low |
| Progress | The user sees what's happening | Low |
| Immediate | The user doesn't wait | High |
| Anticipating | The user doesn't have to ask | High |

The agent that has all 4 is the responsive agent. The
agent that has 1 is the slow agent. The responsive agent
is adopted. The slow agent is not.

## The 80/20

80% of the value comes from:
- Streaming (the user sees the first word)
- Progress (the user sees what's happening)

20% comes from:
- Immediate (the user doesn't wait)
- Anticipating (the user doesn't have to ask)

Focus on the 80% first. Add the 20% as you grow.

## The test

The agent feels responsive if:
- A user can see the first word in 1 second
- A user can see what's happening
- A user can get a response in < 100ms (for simple tasks)
- A user gets what they want without asking

If any of these fails, the agent feels slow. Fix the
agent.

## The lesson

4 ways. 1 responsive. 1 lesson: invest in the small
touches.

The agent that invests in the small touches is the
responsive agent. The agent that doesn't is the slow
agent. The responsive agent is adopted. The slow agent
is not.

The agent era is here. The responsive is the design. The
design is the discipline. The discipline is the speed.
