We've had multiple platform shifts: mainframes → PCs, PCs → web,
web → mobile, mobile → cloud. Each was a 10x opportunity. The
agent era is the next one. What makes it different.

## The pattern

Each platform shift has the same shape:
- A new capability (interactivity, mobility, intelligence)
- A new constraint (CPU, network, latency)
- A new business model (perpetual license, subscription, usage)
- A new set of winners (Microsoft, Google, Apple, AWS)

Each shift creates a 10x opportunity. The companies that get it
right become the next-generation giants. The companies that
miss it become the next-generation has-beens.

The agent era is the next shift. The capability is autonomous
task completion. The constraint is the LLM's reasoning ability.
The business model is usage-based. The winners are still TBD.

## What's different

The agent era is different from the SaaS era in 5 ways:

### 1. The unit of value is the outcome, not the seat

In the SaaS era, the user paid per seat. The more users, the
more revenue. The unit of value was the seat.

In the agent era, the user pays per outcome. The agent charges
per task completed, per issue triaged, per doc updated. The
unit of value is the outcome.

The implication: pricing is harder. You can't just count seats.
You have to measure outcomes. The user only pays if the agent
delivers.

### 2. The marginal cost is non-zero

In the SaaS era, the marginal cost of serving one more user was
near zero. The infrastructure was fixed; the user was free.

In the agent era, the marginal cost of serving one more task is
non-zero. Each task uses LLM tokens. Each token costs money.
The cost scales with usage.

The implication: unit economics matter. You can't just grow
users; you have to make sure each user is profitable.

### 3. The quality is variable

In the SaaS era, the software was deterministic. The same input
produced the same output. Quality was measurable.

In the agent era, the software is non-deterministic. The same
input can produce different outputs. Quality is fuzzy.

The implication: evaluation is harder. You can't just assert
"output === expected." You have to evaluate the quality of the
output, often with another LLM.

### 4. The data is the moat

In the SaaS era, the moat was features. The more features, the
harder to clone.

In the agent era, the moat is data. The agent's memory is its
competitive advantage. The more it knows, the better it is.

The implication: the cold-start problem is real. New agents
have no memory. Established agents have months of accumulated
context. The moat compounds.

### 5. The human is the bottleneck

In the SaaS era, the bottleneck was the software. The user
typed; the software responded.

In the agent era, the bottleneck is the human. The agent
proposes; the human approves. The approval flow is the
bottleneck.

The implication: the agent's value is capped by the human's
ability to review. The faster the human can review, the more
the agent can do.

## The 10x opportunity

Each platform shift created 10x value for the winners. The
agent era is no different. The 10x comes from:

- **More tasks automated.** 10x more issues triaged, 10x more
  docs updated, 10x more code reviewed.
- **Higher quality.** The agent doesn't get tired, doesn't take
  breaks, doesn't forget.
- **Lower cost.** A $0.03 triage agent is cheaper than a $15/hr
  human.
- **Faster iteration.** The agent can try 10 approaches in
  parallel. The human picks the best.

The 10x is real. The companies that capture it will be the next
giants.

## The risk

The agent era has a unique risk: **the agent can act on your
behalf in ways you didn't intend.**

A SaaS app does what the user clicks. An agent does what the
LLM decides. The decision space is larger. The mistakes are
larger.

The mitigation:
- **Approval flows** for high-stakes actions
- **Permission boundaries** for low-stakes actions
- **Audit trails** for everything
- **Reversibility** for reversible actions
- **Bounded agency** for high-stakes domains

The risk is real. The mitigations are partial. Deploy
accordingly.

## The future

The agent era is the biggest platform shift since the web.
The opportunity is 10x. The risk is real. The winners are still
TBD.

The companies that will win:
- **The agent framework** (the "OS" of agents — Husk, LangChain, etc.)
- **The agent host** (the "GitHub" of agents — gitagent, Vercel for agents, etc.)
- **The vertical agent** (the "killer app" of agents — coding, customer support, sales, etc.)
- **The agent infra** (the "AWS" of agents — observability, eval, hosting, etc.)

Each is a multi-billion-dollar opportunity. The first-mover
advantage is real but small. The execution advantage is
everything.

## The meta

I've been building in the SaaS era for 10 years. The agent
era is different. The unit economics are different. The moat
is different. The bottleneck is different.

But the meta-pattern is the same:
- **Ship small.** Iterate. The 10x is in the iteration, not the
  initial commit.
- **Open-source early.** Visibility is leverage.
- **Document the why.** The README is the moat.
- **Automate the boring parts.** Maintenance is the killer.
- **Focus on fewer projects.** Depth beats breadth.

These work in any era. The agent era is no different. The
specifics change; the meta-patterns don't.

The agent era is the next platform shift. The opportunity is

If you're an investor, the agent era is the next 10x. The

The meta: the agent era is here. Build accordingly.
